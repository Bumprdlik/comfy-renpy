⬡ Comfy-Renpy

  1. 1
    Vytvoř Ren'Py projekt

    Otevři Ren'Py launcher → Create New Project. Launcher vygeneruje správnou strukturu včetně gui/, options.rpy a dalších souborů. Výsledek bude vypadat takto:

    MujProjekt/
  game/         ← toto nastav jako gameDir
    gui/
    options.rpy
    script.rpy
  2. 2
    Nastav projekt v comfy-renpy

    Klikni na ⚙ Nastavení a zadej cestu přímo ke složce game/ (ne ke kořeni projektu). Sem půjdou exportované soubory. Editor tě upozorní, pokud cesta nevypadá správně. Volitelně nastav cestu k renpy.exe pro spouštění přímo z editoru.

  3. 3
    Vytvoř lokace a propoj je

    Tlačítkem + Location přidej místnosti. V pravém panelu pojmenuj každou lokaci a přidej exity (směry, kudy se dá odejít). Pak přetáhni drát z exit portu jedné lokace na vstup druhé — tak vzniká mapa.

  4. 4
    Přidej eventy, itemy a postavy

    + Event — scéna nebo událost vázaná na lokaci.
    + Item — předmět ve hře. Každý item dostane pickup label a automaticky se přidá do menu lokace, kde se nachází.
    + Character — postava; při exportu se vygeneruje define řádek do comfy_init.rpy.

  5. 5
    Exportuj — hra je okamžitě hratelná

    Tlačítko Export .rpy vygeneruje kostry souborů. První export vyplní popis lokace a placeholder dialog — hru lze ihned spustit a procházet. Pokud máš Quest uzly, export vytvoří comfy_screens.rpy s tlačítkem Questy N vpravo nahoře.

    Co funguje bez dialogů: pohyb mezi lokacemi, popis místností (z pole description), sebrání itemů, spouštění eventů (placeholder text).

    Co potřebuje body_text: questy se nespustí ani neposunou bez volání comfy_quest_start("id") a comfy_quest_advance("id") v těle eventu. body_text lze napsat přímo v properties panelu (nebo v IDE v sekci kind=body), případně ho vygenerovat přes AI (tlačítko ✨).

  6. 6
    Sleduj postup přes Scan

    Tlačítko Scan zkontroluje stav každého uzlu a zobrazí barevný badge na uzlu.

    written stub missing drift
Location

Reprezentuje místnost nebo oblast ve hře. Uzly Location se propojují hranami, které tvoří mapu hry.

idUnikátní identifikátor lokace, používá se v jump location_{id}. Jen písmena, čísla, podtržítko.
labelZobrazovaný název místnosti (zobrazí se hráči v menu).
descriptionPopis lokace — při prvním exportu se vloží jako narátorská řádka do těla souboru.
exitsSeznam výstupů ve formátu název:cíl_id, např. north:hall. Každý exit vytvoří výstupní port uzlu.
Event

Scéna nebo událost vázaná na konkrétní lokaci. Generuje se do events/{id}.rpy.

idUnikátní identifikátor eventu.
location_idID lokace, kde se event odehrává. Musí odpovídat existující Location.
triggerPodmínka spuštění: auto_enter, menu_choice, condition.
trigger_labelText volby v menu, pokud je trigger menu_choice.
prerequisiteRen'Py podmínka (Python výraz), která musí být splněna pro zobrazení eventu.
timeHerní čas: any, day, night, apod.
repeatableZda se event může spustit opakovaně (true/false).
priorityČíslo priority při více splněných eventech najednou (vyšší = dřív).
notesVolné poznámky pro tvůrce — exportuje se jako komentář.
Item

Předmět ve hře. Exportuje se jako pickup label do items/{id}.rpy. V menu lokace (kde je přiřazen) se automaticky přidá volba Sebrat: {název} hlídaná funkcí comfy_has().

idUnikátní identifikátor itemu — použije se jako comfy_give("{id}").
nameZobrazovaný název v menu lokace.
descriptionPopis předmětu — vloží se jako dialog při sebrání.
location_idID lokace, kde lze předmět sebrat.
pickup_conditionVolitelná Python podmínka pro sebrání (AND s not comfy_has), např. comfy_quest_active("q1") and comfy_quest_stage("q1") >= 2. Prázdné = vždy sebratelný.
Character

Postava ve hře. Při exportu se automaticky vygeneruje define {id} = Character("{name}") do comfy_init.rpy.

idUnikátní identifikátor postavy — použije se jako název proměnné v Ren'Py.
nameJméno postavy zobrazované v dialogovém boxu.
voiceHlasový tag pro voice acting (voice_tag=).
sprite_idID spritu postavy pro show {sprite_id}.
Quest

Quest / úkol. Generuje se do quests/{id}.rpy s proměnnými pro sledování postupu.

idUnikátní identifikátor questu — použije se jako prefix proměnných ({id}_active, {id}_stage).
titleZobrazovaný název questu.
descriptionPopis questu pro tvůrce.
stagesSeznam fází — každá fáze na samostatném řádku. Volitelně přidej hint za |, např. Promluvit s Elarou | Sedí v hospodě. Hint se zobrazí v quest logu ve hře.

Export .rpy

Tlačítko Export .rpy projde všechny uzly a vygeneruje (nebo aktualizuje) .rpy soubory. Hra je po prvním exportu okamžitě hratelná.

Soubory se ukládají do:

  • {gameDir}/locations/{id}.rpy — Location uzly
  • {gameDir}/events/{id}.rpy — Event uzly
  • {gameDir}/items/{id}.rpy — Item uzly (pickup labely)
  • {gameDir}/quests/{id}.rpy — Quest uzly
  • {gameDir}/comfy_init.rpy — inventář + Character define řádky

Pokud gameDir není nastaven, exportuje do output/ vedle serveru.

Marker systém

Každý soubor má tři sekce ohraničené markery:

# [COMFY-START id=kitchen kind=header]
label location_kitchen:
# [COMFY-END]

# [COMFY-START id=kitchen kind=body]
    "V kuchyni voní čerstvá káva."  <-- tvůj dialog / AI obsah
# [COMFY-END]

# [COMFY-START id=kitchen kind=exits]
    menu:
        "north":
            jump location_hall
        "Sebrat: Klíč" if not comfy_has("klic"):
            call item_klic
    jump location_kitchen
# [COMFY-END]

Sekce body Comfy-Renpy při re-exportu nikdy nepřepíše — piš nebo generuj AI obsah tam. Sekce header a exits se aktualizují automaticky.

AI generování dialogů

V properties panelu eventu nebo lokace klikni na ✨ Generovat. AI vytvoří dialog podle popisu uzlu, postav a itemů v lokaci. Tlačítkem 💾 Zapsat do souboru se dialog zapíše přímo do sekce kind=body — bez nutnosti kopírovat do IDE.

Scan — stavy uzlů

writtenSoubor existuje a má dialogové řádky (uvozovky, jump, show, menu…).
stubSoubor existuje, marker je přítomen, ale žádný dialog zatím není napsán.
missingSoubor pro tento uzel neexistuje — spusť Export.
driftSoubor existuje bez COMFY markeru, nebo soubor existuje pro uzel, který v grafu chybí (orphan).
